**ID:** US_07.003 | Organization folder Configuration > Change Scan Organization Folder Triggers

**Description:**

**As a** user,
**I want to** configure scan triggers for an **Organization Folder item**,
**So that I can** control how often Jenkins scans the GitHub organization for repository changes and automatically discovers new repositories.

**Preconditions**

1. Jenkins instance is up and running and accessible to the user.
2. User is authenticated and has permissions to configure jobs.
3. User has already created an **Organization Folder** item and is on the **Configuration** page.
4. User is viewing the **Scan Organization Folder Triggers** section of the **Configuration** page.


**Acceptance Criteria:**

1. In the **Scan Organization Folder Triggers** section, the **Periodically if not otherwise run** checkbox is enabled by default; this option schedules a periodic fallback indexing when no automatic indexing has occurred within the specified interval.
2. **Interval**. Select a value from the dropdown list to specify the maximum allowed time since the last indexing before a new indexing is triggered. Available options: 
    - 1 minute, 
    - 2 minutes, 
    - 3 minutes, 
    - 5 minutes, 
    - 10 minutes, 
    - 15 minutes, 
    - 20 minutes, 
    - 25 minutes, 
    - 30 minutes, 
    - 1 hour, 
    - 2 hours, 
    - 4 hours, 
    - 8 hours, 
    - 12 hours, 
    - 1 day, 
    - 2 days, 
    - 1 week, 
    - 2 weeks, 
    - 4 weeks.
