**ID:** US_01.005 | New Item > Create Multibranch Pipeline

**Description:**
**As a** user,
**I want to** create a Multibranch Pipeline item that automatically discovers and builds branches from my Git repository, 
**So that I can** manage CI/CD for multiple branches without creating separate jobs manually.

**Preconditions**
- Jenkins instance is up and running and accessible to the user
- User is authenticated and has permissions to create new items. 
- User has navigated to the New Item creation page (reached via "New Item" link on the dashboard — note: there's a "+" icon next to the "New Item" label, or by the "Create a job" button if you're starting Jenkins for the first time). 
- Entered the unique folder name.

**Acceptance Criteria:**

1. By clicking the "New Item" (with a "+" icon) link on the left navigation menu of the Jenkins dashboard,
the user is redirected to the item creation page (http://localhost:8080/view/all/newJob), where the following item types are available as selectable options:
- Pipeline
- Freestyle project
- Multi-configuration project
- Folder
- Multibranch Pipeline
- Organization Folder

2. While on the item creation page, the user can:
- Enter a valid and unique item name (alphanumeric, hyphen, underscore — no spaces or special characters).
- Select **Multibranch Pipeline** as the item type.
- Click "OK" only after both a valid name and **Multibranch Pipeline** type are provided.
- Return to the Jenkins dashboard without creating an item — by clicking the "Jenkins"  text with the logo in the top-left corner.
3. After the user enters a valid unique item name, selects an item type **Multibranch Pipeline**, and clicks "OK", the user is redirected to the **Multibranch Pipeline** configuration page item configuration page.
