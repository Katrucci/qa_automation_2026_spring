**ID:** US_06.009| Multibranch pipeline Configuration > Change Health metrics

**Description:**

**As a** user,
**I want to** configure Health metrics for a Multibranch Pipeline project,
**So that I can** visualize the health status of branches based on build stability, test results, and other metrics.

**Preconditions**

1. Jenkins instance is up and running and accessible to the user.
2. User is authenticated and has permissions to configure jobs.
3. User has already created a Multibranch Pipeline item and is on the configuration page.
4. User is viewing the **Health metrics**.

**Acceptance Criteria:**

1. In the **Health metrics** section, the **Health metrics** dropdown is displayed and an **Add** button is available.
2. Clicking the **Add** button opens a list with the following options: 
    - **Child item with the given name**; 
        - Child Name. Specify the name of the child item (project, folder, or other object) within this folder whose health status will represent the overall health of this folder. Jenkins will monitor the health of the specified child item and use its metrics to display the folder's aggregated health status. 
        - The **Add metrics** button lets you add the following two items:Child item with worst health, Health of the primary branch of a repository.
    - **Child item with worst health**. 
        - **Recursive** — checkbox selected by default. Enable this option to consider items within sub-folders (nested folders) when determining the worst health status for this folder. If disabled, only direct child items at the current folder level will be evaluated. When enabled, Jenkins recursively traverses all sub-folders and their children to find the item with the most severe health condition, and uses that health status to represent the overall health of the parent folder. 
        - The **Add metrics** button lets you add the following two items:Child item with the given name, Health of the primary branch of a repository.
    - **Health of the primary branch of a repository**.This metric reports the health status of only the primary branch (e.g., main, master, trunk) for source control systems that support the concept of a primary branch. If a primary branch exists, Jenkins evaluates its health and uses it as the representative health metric for the repository, ignoring other non-primary branches. 
        - The **Add metrics** button lets you add the following two items: Child item with the given name, Child item with worst health
  
`When the three metrics — Child item with the given name, Child item with worst health, and Health of the primary branch of a repository — are selected, the Add metrics button becomes disabled.`
