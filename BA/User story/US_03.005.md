**ID:** US_03.005 | Pipeline Configuration > Advanced

**Description:**
**As a** user,
**I want to** configure advanced settings for a **Pipeline item**,
**So that I can** fine-tune Pipeline behavior for complex CI/CD workflows.

**Preconditions**
1. Jenkins instance is up and running and accessible to the user.
2. User is authenticated and has permissions to configure jobs.
3. User has already created a **Pipeline item** and is on the configuration page.
4. User is viewing the **Advanced** section of the Pipeline configuration.

**Acceptance Criteria:**

1. By navigating to the **Pipeline configuration** page, the user is presented with the **Advanced** button — displayed on the left with a wrench icon, after clicking the **Advanced** button, the Advanced section opens.
2. Clicking on dropdown button **Advanced** opens Dropdown menu, where the following configuration options are available:
- Enter a custom **Quiet period** value (Number of seconds, e.g., 5) to delay newly triggered builds in the queue before execution.  
  *(Note: If a new build is triggered while another build is already waiting in the queue with a quiet period active, the quiet period will not be reset. The new build will not be added to the queue unless the project is parameterized and has different parameters.)*.
- Enter a **Display Name** (e.g., `My Deployment Pipeline`) to show a friendly, non-unique name for the project throughout the Jenkins web GUI.  
  *(If left empty, Jenkins defaults to showing the project name.)*.

3. After configuring Advanced settings and clicking **Save**, the user is redirected to the Pipeline project page where appears the Display Name from Advanced (e.g., `My Deployment Pipeline`).
